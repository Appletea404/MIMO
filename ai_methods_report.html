① KoNLPy Okt — 한국어 형태소 분석기

형태소 분석이란?

형태소(形態素)는 의미를 가진 가장 작은 언어 단위예요.
"검버섯이 생겼어요"를 그냥 띄어쓰기로 나누면 ["검버섯이", "생겼어요"] 가 되는데, 이건 컴퓨터가 검색하기 어려워요.
형태소 분석을 하면 ["검버섯(명사)", "생기다(동사)"] 처럼 의미 있는 핵심 단어만 추출할 수 있어요.

Okt(Open Korean Text)란?

트위터에서 만든 오픈소스 한국어 형태소 분석기예요. KoNLPy 라이브러리를 통해 사용해요.
내부적으로 Java(자바)를 실행해서 동작하기 때문에 JDK 설치가 필수예요.
처음 실행 시 JVM(자바 가상머신)을 켜는 데 5~15초 걸려요.

이 프로젝트에서 하는 일 (4단계)

품사(POS) 태그 종류

태그의미예시이 프로젝트에서
Noun명사검버섯, 여드름, 피부✅ 사용
Verb동사생기다, 없애다, 개선하다✅ 사용
Adjective형용사건조하다, 민감하다✅ 사용
Josa조사이, 가, 을, 를, 은, 는❌ 제거
Eomi어미~었어요, ~습니다❌ 제거

stem=True 옵션이란?

단어를 기본형으로 바꿔줘요.
"생겼어요" → "생기다" / "없애줘요" → "없애다" / "건조했어요" → "건조하다"
이렇게 하면 같은 의미의 다양한 표현이 같은 단어로 통일돼서 비교 정확도가 올라가요.

불용어(Stopwords)란?

분석에 도움이 안 되는 단어 목록이에요.
"나", "우리", "아", "정말" 같은 단어는 어떤 문장에나 나오기 때문에 유사도 계산에 방해가 돼요.
datasets/stopwords.csv에 목록이 저장되어 있고, 파일이 없으면 기본 불용어 18개를 사용해요.

② TF-IDF — 텍스트 벡터화

TF-IDF란?

TF-IDF(Term Frequency - Inverse Document Frequency)는 단어의 중요도를 숫자로 표현하는 방법이에요.
쉽게 말하면 "이 단어가 이 문서에서 얼마나 특별한가" 를 점수로 매기는 거예요.

TF (단어 빈도, Term Frequency)

TF = 이 문서에서 단어가 나온 횟수 / 전체 단어 수
문서: "검버섯 검버섯 피부 관리" → "검버섯" TF = 2/4 = 0.5

IDF (역문서 빈도, Inverse Document Frequency)

IDF = log(전체 문서 수 / 이 단어가 나온 문서 수)
"피부"는 9031개 중 8000개 문서에 등장 → IDF 낮음 (흔한 단어)
"검버섯"은 9031개 중 50개 문서에만 등장 → IDF 높음 (희귀한 단어 = 중요!)
TF-IDF(단어) = TF × IDF
희귀하면서 자주 나오는 단어 = 높은 점수 → 이 문서를 대표하는 핵심 단어

왜 "피부"보다 "검버섯"이 더 중요할까?

❌ "피부" — TF-IDF 낮음

9031개 문서 거의 모두에 등장
→ IDF 값이 매우 낮음
→ 어떤 문서와도 비슷해 보임
→ 구별에 도움이 안 됨

✅ "검버섯" — TF-IDF 높음

검버섯 관련 소수 문서에만 등장
→ IDF 값이 높음
→ 검버섯 문서를 딱 집어서 찾음
→ 유사도 계산에 결정적 역할

③ 코사인 유사도 — 유사도 계산

코사인 유사도란?

두 벡터가 얼마나 같은 방향을 가리키는지로 유사도를 측정하는 방법이에요.
결과값은 0 ~ 1 사이로, 1에 가까울수록 비슷하고 0에 가까울수록 다른 내용이에요.

왜 "거리"가 아닌 "각도"로 비교할까?

"검버섯"이 10번 나온 긴 문서와 2번 나온 짧은 문서를 비교할 때, 거리로 비교하면 같은 내용인데 다르다고 나와요.
각도로 비교하면 두 문서가 같은 단어를 비슷한 비율로 쓰면 같다고 판단해요.
즉 문서 길이에 영향을 받지 않아요.
코사인 유사도 = (A · B) / (|A| × |B|)
A · B = 두 벡터의 내적 / |A|, |B| = 각 벡터의 크기(길이)

linear_kernel 함수를 쓰는 이유

sklearn의 linear_kernel()은 TF-IDF 벡터에 대해 코사인 유사도를 빠르게 계산해줘요.
TF-IDF 벡터는 이미 정규화(길이=1)가 되어 있어서 내적(dot product)만 계산해도 코사인 유사도와 같아요.
9031개를 동시에 계산해서 결과를 배열로 한 번에 반환하기 때문에 빠른 거예요.

한계점

이 세 가지 방식의 한계
  • 문맥을 이해하지 못해요: "검버섯이 없어졌어요"와 "검버섯이 생겼어요"를 비슷하다고 판단할 수 있어요 (둘 다 "검버섯" 단어 포함)
  • 동의어를 모르:요 "기미"와 "검버섯"이 비슷한 의미인데 다른 단어로 인식해요
  • 신조어에 약해요: 학습 데이터에 없는 새로운 단어는 벡터 점수가 0이에요
이 한계를 극복하려면 BERT, KoBERT, Sentence-Transformers 같은 딥러닝 기반 모델을 사용해야 해요. 문맥과 의미를 이해할 수 있지만 속도가 느리고 GPU가 필요해요.
