2. 서버 시작 흐름

서버가 켜지는 순간 아래 순서대로 자동 실행돼요. 이 과정이 끝나야 요청을 받을 수 있어요.
① Flask 앱 생성 + 비밀키 로드 (SECRET_KEY)
↓
② 피부 상담 데이터 로드 (skin_data_final.csv + rebuild TF-IDF 모델)
↓
③ 올리브영 제품·리뷰 데이터 로드 시도 (실패해도 서버는 정상 가동)
↓
④ KoNLPy Okt 형태소 분석기 초기화 (JVM 시작 — 5~15초 소요)
↓
⑤ 불용어 목록 로드
↓
⑥ DB 초기화 (users.db 테이블 생성 + 관리자 계정 자동 생성)
↓
✅ 서버 준비 완료 — 요청 대기

관리자 계정 기본값

항목기본값변경 방법

서버 실행 명령어

4. 추천 알고리즘 흐름

🔬 Expert Mode (전문가 상담 기반)

피부 고민 키워드를 입력하면 9031개 상담 데이터에서 가장 유사한 답변을 찾아요.

⭐ Review Mode (리뷰 기반 제품 추천)

올리브영 리뷰 데이터에서 고민과 관련된 실제 제품 3개를 추천해요.

Expert Mode 상세 흐름

📍 키워드 있을 때

① 사용자 입력 수신 (keyword, gender, age, skin_type)
↓
② 텍스트 전처리 — 한글만 추출 → 형태소 분석(Okt) → 명사/동사/형용사만 남김
↓
③ TF-IDF 변환 — 전처리된 텍스트를 숫자 벡터로 변환
↓
④ 코사인 유사도 계산 — 사용자 벡터 vs 9031개 데이터 벡터
↓
⑤ 프로필 보너스 추가 — 성별/연령/피부타입 일치 시 각 +0.05점 (최대 +0.15)
↓
⑥ 최종 점수 최고값 행 선택 → 응답 반환

📍 키워드 없을 때 (프로필만 선택)

프로필 조건으로
데이터 필터링
→
필터된 첫 번째
행 반환

Review Mode 상세 흐름

키워드 전처리
→
리뷰 TF-IDF
유사도 계산
→
유사도 높은 순
정렬
→
중복 제거 후
상위 3개 제품 반환
Review Mode는 올리브영 데이터가 로드된 경우에만 사용 가능해요. 데이터가 없으면 503 오류를 반환해요.

프로필 보너스 원리

조건보너스 점수이유
성별 일치+0.05같은 성별 데이터 우선 선택
연령대 일치+0.05같은 연령 데이터 우선 선택
피부 타입 일치+0.05같은 피부 타입 우선 선택
최대 보너스+0.153가지 모두 일치 시
보너스가 너무 크면 내용과 무관한 데이터가 선택될 수 있어서 0.05로 작게 설정했어요. 내용 유사도가 주점수, 프로필은 동점일 때 타이브레이커 역할이에요.

5. 사용자 인증 흐름

회원가입

폼 입력 (아이디, 이름, 이메일, 비밀번호, 프로필)
↓
CSRF 토큰 검증
↓
입력값 유효성 검사
↓
비밀번호 해시 생성 (werkzeug)
↓
DB 저장 → 로그인 페이지로 이동

로그인

아이디 + 비밀번호 입력
↓
CSRF 토큰 검증
↓
DB에서 사용자 조회
↓
비밀번호 해시 비교
↓
세션에 user_id 저장 → 메인으로 이동

회원가입 유효성 검사 규칙

항목규칙
아이디영문 소문자·숫자·밑줄만, 4~20자
이름2~30자
이메일@와 도메인 포함 형식
비밀번호영문 + 숫자 포함, 8자 이상

모든 요청 전 실행 (before_request)

모든 요청마다 세션의 user_id로 DB에서 사용자 정보를 조회해서 g.user에 저장해요. 로그인 여부와 관리자 여부를 판단하는 데 사용돼요.

7. 보안 처리

보안 항목방법목적
CSRF 방어폼마다 csrf_token 검증 (compare_digest)외부에서 위조 요청 차단
비밀번호werkzeug generate_password_hash평문 저장 방지
세션 보안HTTPONLY + SameSite=Lax 쿠키XSS·CSRF 방어
세션 유지8시간 (PERMANENT_SESSION_LIFETIME)자동 로그아웃
비밀키파일 또는 환경변수에서 로드세션 위변조 방지
안전한 리다이렉트같은 도메인만 허용 (is_safe_redirect)Open Redirect 방어
제품 URL 검증oliveyoung.co.kr 도메인만 허용악성 링크 차단
입력 길이 제한keyword 최대 500자과도한 입력 방어
